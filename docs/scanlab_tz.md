# Техническое задание (ТЗ) на разработку сайта SCAN.LAB

Дата: 2026-02-17<br>
Язык: RU (UTF-8)

## 1. Общие сведения

**SCAN.LAB** — высокотехнологичный сервис 3D-сканирования и 3D-печати для B2B и B2C сегментов (музеи, промышленность, киноиндустрия).

**Цель сайта**

- Демонстрация экспертности
- Презентация портфолио выполненных работ
- Генерация лидов

**Визуальный стиль**

- Технологичный dark mode
- Минимализм
- Акцентный цвет: Electric Blue `#0066FF`

## 2. Структура сайта и навигация

Сайт представляет собой многостраничное решение с единой навигационной панелью (**Sticky Header**).

### 2.1. Информационная архитектура

- Главная страница (лендинг): Hero-секция, Услуги, О компании, Процесс работы, FAQ.
- Портфолио: общая сетка проектов с фильтрацией.
- Страница проекта (кейс): подробное описание конкретной работы.
- Контакты: форма обратной связи, карта, прямые контакты.

### 2.2. Навигация (Header)

Состав:

- Слева: логотип SCAN.LAB (SVG).
- Центр: ссылки на разделы: Портфолио, Контакты.
  - Блоки «Услуги» и «Процесс» присутствуют на главной странице как секции лендинга и не дублируются отдельными пунктами глобального меню.
- Справа: CTA:
  - primary: «Написать в Telegram» (Electric Blue)
  - secondary: «Рассчитать стоимость» (outline)

Поведение:

- Липкая шапка (Sticky).
- При скролле добавляется эффект Glassmorphism (размытие фона).

## 3. Описание страниц и функциональных блоков

### 3.1. Главная страница (основные сущности)

**Hero Section**

- Заголовок H1
- Краткое УТП
- Фоновое видео или высококачественное изображение процесса сканирования
- Статистические плашки:
- «Точность до 0.05мм»
- «Разрешение 8K»

**Блок услуг (Grid)**

- 4 карточки
- Каждая карточка:
- Иконка (line-art)
- Заголовок
- Описание
- Цена «от...»

**Блок «Наш процесс»**

- Интерактивная временная шкала из 7 шагов:
- 1) Анализ ТЗ
- 2) Договор
- 3) Сканирование
- 4) Обработка данных
- 5) 3D-моделирование
- 6) Контроль качества
- 7) Выдача файлов

**FAQ**

- Аккордеон (например, `<details>` с кастомной стилизацией)
- Мягкие анимации открытия/закрытия

### 3.2. Страница «Портфолио»

**Сетка**

- Mixed Grid (динамическая сетка с разными размерами карточек)

**Фильтрация**

- Категории: «Все проекты», «Архитектура», «Музей», «Промышленность», «Арт».
- Без перезагрузки страницы
- Мягкая анимация (Fade/Scale) при переключении категорий

**Hover-эффект карточек**

- Появление кнопки «Смотреть кейс»
- Легкое свечение (Glow) границ

### 3.3. Страница кейса (Case Study)

**Интерактив**

- Слайдер «До/После» (Before/After Slider) для сравнения фотографии объекта и полученной 3D-сетки

**Технические параметры**

- Боковая панель с данными:
- Заказчик
- Срок
- Точность
- Оборудование

**CTA блок**

- Секция «Заинтересовал проект?» с кнопкой перехода в Telegram

### 3.4. Страница «Контакты»

- Форма обратной связи (лидогенерация)
- Карта (embed/виджет)
- Прямые контакты (телефон, email, Telegram и т.п.)

## 4. Перечень сущностей и элементов (UI Kit)

### 4.1. Типографика

- Заголовки (H1-H3): Inter или Roboto, Weight 700 (Bold), цвет `#FFFFFF`.
- Основной текст: Inter, Weight 400 (Regular), цвет `#A0A0A0`.

### 4.2. Цветовая палитра

- Background: `#0A0A0A` (Deep Black)
- Surface: `#1A1A1A` (Dark Grey для карточек)
- Accent: `#0066FF` (Electric Blue)
- Lines/Borders: `#2E2E2E`

### 4.3. Компоненты форм

**Input fields**

- Темный фон
- Тонкая рамка
- При фокусе: синее свечение

**Dropdown**

- Стилизованный список выбора услуг в общей теме

**Кнопки**

- Primary: заливка Electric Blue, текст White
- Secondary: прозрачная, белая или синяя рамка

## 5. Технические требования

- Адаптивность: Desktop (1440px+), Tablet (768px), Mobile (375px).
- Анимация: плавное появление элементов (Fade In) при скролле, микро-взаимодействия при наведении на кнопки и карточки.
- Производительность:
- оптимизация тяжелых изображений и 3D-рендеров (WebP/AVIF)
- lazy loading для сетки портфолио

## 6. Требования к контенту

- Иконки: только векторный формат (SVG).
- Изображения: рендеры высокого разрешения, демонстрирующие сетку (wireframe) объектов.
- Локализация: полная поддержка русского языка (кодировка UTF-8).

## 7. Спецификация раздела «Портфолио»

Раздел предназначен для демонстрации возможностей сканирования на объектах разного масштаба. Дизайн базируется на динамической сетке (**Mixed Grid**) в темной теме (`#0A0A0A`).

### 7.1. Система фильтрации и навигации

- Категории: «Все проекты», «Архитектура», «Музей», «Промышленность», «Арт».
- Тип фильтрации: мягкая анимация (Fade/Scale) при переключении категорий без перезагрузки страницы.
- Статистика раздела: инфо-панель сверху с общими счетчиками (например: «1,240+ моделей оцифровано», «Точность 0.05 mm»).

### 7.2. Типология карточек в сетке (Mixed Grid)

Для создания визуального ритма используются 3 формата карточек.

**Флагманская (Wide/Hero)**

- Размер: 2x1 или 2x2 ячейки.
- Особенности: включает интерактивный микро-слайдер «До/После» прямо в превью.
- Поля:
- Заголовок (H3)
- Развернутое подзаглавие
- Теги технологий (например: LIDAR, OBJ)

**Портретная (Vertical)**

- Размер: 1x2 ячейки.
- Назначение: оптимальна для статуй, колонн и высоких вертикальных объектов.
- Поля:
- Категория в верхнем углу (бейдж)
- Название в нижней трети

**Стандартная (Square)**

- Размер: 1x1 ячейка.
- Назначение: для деталей, запчастей или небольших артефактов.
- Поля:
- Центрированное изображение объекта на темном градиентном фоне

### 7.3. Поля и бейджи карточек

- Заголовок: название объекта (Inter Bold, 18-22px).
- Бейджи категорий: закругленные плашки (Radius 4px) с полупрозрачным фоном. Цвет текста совпадает с Electric Blue (`#0066FF`).
- Технические параметры: мелкие подписи внизу карточки (например: «Точность: 0.05mm», «Полигонов: 15M»).
- Форматы файлов: список доступных форматов (STEP, STL, FBX) в виде компактных тегов.

## 8. Спецификация страницы «Детали проекта» (Кейс)

Страница-лонгрид, разделенная на смысловые блоки для глубокого погружения в технический процесс.

### 8.1. Hero-блок (заголовок и параметры)

- Навигация: хлебные крошки (Главная > Архитектура > Проект).
- Заголовок: крупный H1 с типом услуги (например, «Оцифровка Триумфальных ворот»).
- Параметры проекта (Sidebar):
- Заказчик: логотип или название компании.
- Срок исполнения: количество дней/часов.
- Точность: итоговое отклонение в мм.
- Форматы: с какими данными работает заказчик.
- Кнопка: «Скачать спецификацию (PDF)» для b2b-клиентов.

### 8.2. Интерактивный блок «Фото / Облако точек»

- Функционал: полноэкранный слайдер с центральным манипулятором.
- Контент: слева — реальное фото объекта; справа — результат сканирования (полигональная сетка или текстурированная модель).
- Подпись: инструкция «Перетащите слайдер, чтобы сравнить».

### 8.3. Блок «Технические детали» (оборудование и ПО)

Представлен в виде таблицы или набора карточек.

- Оборудование: список использованных сканеров (например, FARO Focus S, Artec Leo) с иконками чипов.
- ПО: программы для сшивки и ретопологии (например, RealityCapture, ZBrush).

### 8.4. Результаты и процесс

- Текстовый блок: описание задачи, этапов сканирования (ночная съемка, использование подъемников) и итогового решения.
- Галерея результатов: сетка 3D-рендеров отдельных фрагментов объекта с максимальной детализацией.

### 8.5. Футер кейса (рекомендации и CTA)

- Секция «Другие проекты»: карусель из 3-х карточек смежной тематики.
- Лид-магнит: горизонтальная плашка «Заинтересовал подобный проект?» с кнопкой «Обсудить в Telegram».
